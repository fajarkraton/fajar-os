
## Chapter 1: Hello from Bare Metal

The simplest bare-metal program — print to UART and halt.

### `hello.fj`

```fajar
@panic_handler
fn panic(code: i64) -> i64 { code }

fn fj_exception_sync(esr: i64, elr: i64, far: i64) {}
fn fj_exception_irq() {}

fn kernel_main() -> i64 {
    println("Hello from bare metal!")
    println("Running on ARM64 with no OS underneath.")
    0
}

fn main() -> i64 { 0 }
```

### Build & Run

```bash
fj build hello.fj --target aarch64 --no-std -o hello.elf

qemu-system-aarch64 \
  -M virt,gic-version=3 \
  -cpu cortex-a76 \
  -m 512M \
  -kernel hello.elf \
  -nographic
```

### What happens

1. **QEMU** loads `hello.elf` at `0x40000000`
2. **Startup assembly** (generated by Fajar Lang) sets stack, exception vectors, calls `kernel_main`
3. `println("Hello...")` writes string bytes to UART at `0x09000000`
4. The string lives in `.rodata` — no heap allocation needed

### Key concepts

- `@panic_handler`: required for bare-metal — handles traps
- `fj_exception_sync`: handles synchronous exceptions (SVC, faults)
- `fj_exception_irq`: handles hardware interrupts (timer, devices)
- `kernel_main`: your entry point (called by startup code)
- `println()`: works in bare-metal via UART MMIO
- `--target aarch64 --no-std`: bare-metal ARM64, no standard library

## Chapter 2: UART — Reading and Writing

Direct hardware register access via `volatile_read`/`volatile_write`.

### `uart.fj`

```fajar
@panic_handler
fn panic(code: i64) -> i64 { code }
fn fj_exception_sync(esr: i64, elr: i64, far: i64) {}
fn fj_exception_irq() {}

// PL011 UART registers (QEMU virt)
const UART_BASE: i64 = 0x09000000
const UART_DR: i64 = 0x09000000    // Data register
const UART_FR: i64 = 0x09000018    // Flag register (bit 4 = RXFE)

fn putc(c: i64) {
    volatile_write(UART_BASE, c)
}

fn getc() -> i64 {
    // Wait for data
    let mut ready = 0
    while ready == 0 {
        let fr = volatile_read_u32(UART_FR)
        if (fr & 16) == 0 {
            ready = 1
        }
        asm!("wfi")  // sleep until interrupt
    }
    volatile_read_u32(UART_DR) & 0xFF
}

fn kernel_main() -> i64 {
    println("UART Echo — type characters:")
    let mut running = 1
    while running == 1 {
        let fr = volatile_read_u32(UART_FR)
        if (fr & 16) == 0 {
            let c = volatile_read_u32(UART_DR) & 0xFF
            putc(c)  // echo back
            if c == 13 { putc(10) }  // CR → LF
        } else {
            asm!("wfi")
        }
    }
    0
}

fn main() -> i64 { 0 }
```

### Key concepts

- `volatile_read/write`: MMIO register access (not optimized away)
- `volatile_read_u32`: 32-bit register read (ARM peripherals are 32-bit)
- `asm!("wfi")`: Wait For Interrupt — saves power while idle
- PL011 UART: standard ARM serial port, used by QEMU

## Chapter 3: Timer Interrupts

Set up the ARM Generic Timer to fire every 10ms.

### `timer.fj`

```fajar
@panic_handler
fn panic(code: i64) -> i64 { code }

fn fj_exception_sync(esr: i64, elr: i64, far: i64) {}

fn fj_exception_irq() {
    let irq = gic_ack()
    if irq == 30 {
        // Timer fired! Rearm for next 10ms
        timer_set(625000)  // 62.5MHz / 100 = 625000 ticks = 10ms
        putc(46)           // print '.'
    }
    gic_eoi(irq)
}

fn putc(c: i64) { volatile_write(0x09000000, c) }

fn init_gic() {
    // GICv3 Distributor
    let gicd = 0x08000000
    volatile_write_u32(gicd, 0)                    // disable
    let mut i = 1
    while i < 32 {
        volatile_write_u32(gicd + 0x0180 + i * 4, 0xFFFFFFFF)  // disable all
        volatile_write_u32(gicd + 0x0080 + i * 4, 0xFFFFFFFF)  // group 1
        i = i + 1
    }
    volatile_write_u32(gicd, 2)                    // enable group 1

    // GICv3 Redistributor
    let gicr = 0x080A0000
    volatile_write_u32(gicr + 0x0014, volatile_read_u32(gicr + 0x0014) & 0xFFFFFFFD)
    volatile_write_u32(gicr + 0x10080, 0xFFFFFFFF)  // group 1
    volatile_write_u32(gicr + 0x10180, 0xFFFFFFFF)  // disable all

    // CPU interface
    gic_cpu_init(0xFF)
    irq_enable()

    // Enable timer PPI (IRQ 30)
    let bit = 1 << 30
    volatile_write_u32(gicr + 0x10100, bit)
    volatile_write_u8(gicr + 0x10400 + 30, 0x20)  // priority
}

fn kernel_main() -> i64 {
    init_gic()
    println("Timer demo — dots appear every 10ms:")

    // Start timer
    let freq = timer_freq()
    let quantum = freq / 100   // 10ms
    timer_set(quantum)

    // Main loop — WFI wakes on timer interrupt
    let mut count = 0
    while count < 500 {
        asm!("wfi")
        count = count + 1
    }
    println("")
    println("500 timer ticks done!")
    0
}

fn main() -> i64 { 0 }
```

### Key concepts

- `gic_ack()` / `gic_eoi()`: acknowledge and end-of-interrupt for GICv3
- `timer_set(ticks)`: arms the ARM physical timer (CNTP_TVAL_EL0)
- `timer_freq()`: reads counter frequency (CNTFRQ_EL0)
- IRQ 30: non-secure physical timer PPI on GICv3
- `fj_exception_irq`: called automatically when IRQ fires
- The timer fires, handler runs, process resumes after `wfi`

## Chapter 4: Processes and Context Switching

Create multiple processes and switch between them.

### `scheduler.fj`

```fajar
@panic_handler
fn panic(code: i64) -> i64 { code }

fn fj_exception_sync(esr: i64, elr: i64, far: i64) {}

// Process table at 0x47000000 (256 bytes per process)
// [0]=PID, [8]=State, [16]=SP, [24]=entry
const PROC_BASE: i64 = 0x47000000
const CUR_PID: i64 = 0x47001000

fn fj_exception_irq() {
    let irq = gic_ack()
    if irq == 30 {
        timer_set(625000)
        // Round-robin scheduler
        let cur = sched_read_proc(CUR_PID)
        let saved_sp = sched_get_saved_sp()
        if saved_sp > 0 {
            // Save current SP
            sched_write_proc(PROC_BASE + cur * 256 + 16, saved_sp)
            let cur_state = sched_read_proc(PROC_BASE + cur * 256 + 8)
            if cur_state == 2 { sched_write_proc(PROC_BASE + cur * 256 + 8, 1) }

            // Find next ready process
            let mut next = cur + 1
            let mut found = 0
            let mut checked = 0
            while checked < 4 {
                if next > 3 { next = 0 }
                if sched_read_proc(PROC_BASE + next * 256 + 8) == 1 {
                    found = 1
                    checked = 4
                } else { next = next + 1 checked = checked + 1 }
            }
            if found == 1 {
                sched_write_proc(PROC_BASE + next * 256 + 8, 2)
                sched_write_proc(CUR_PID, next)
                sched_set_next_sp(sched_read_proc(PROC_BASE + next * 256 + 16))
            }
        }
    }
    gic_eoi(irq)
}

fn process_a() -> i64 {
    let mut i = 0
    while i < 200 { putc(65) let mut j = 0 while j < 30000 { j = j + 1 } i = i + 1 }
    sched_write_proc(PROC_BASE + sched_read_proc(CUR_PID) * 256 + 8, 4)
    asm!("wfi")
    0
}

fn process_b() -> i64 {
    let mut i = 0
    while i < 200 { putc(66) let mut j = 0 while j < 30000 { j = j + 1 } i = i + 1 }
    sched_write_proc(PROC_BASE + sched_read_proc(CUR_PID) * 256 + 8, 4)
    asm!("wfi")
    0
}

fn putc(c: i64) { volatile_write(0x09000000, c) }

fn create_proc(pid: i64, entry: i64) {
    let base = PROC_BASE + pid * 256
    sched_write_proc(base, pid)
    sched_write_proc(base + 8, 1)  // READY
    let stack_top = 0x48000000 + (pid + 1) * 0x10000 - 16
    let frame_sp = stack_top - 272
    let mut off = 0
    while off < 272 { sched_write_proc(frame_sp + off, 0) off = off + 8 }
    sched_write_proc(frame_sp + 248, entry)  // ELR = entry point
    sched_write_proc(frame_sp + 256, 5)      // SPSR = EL1h
    sched_write_proc(base + 16, frame_sp)
}

fn kernel_main() -> i64 {
    // Init GIC + timer (same as Chapter 3)
    // ... (init_gic code here) ...

    // PID 0 = kernel
    sched_write_proc(PROC_BASE + 8, 2)  // RUNNING
    sched_write_proc(CUR_PID, 0)

    // Create processes
    create_proc(1, fn_addr(process_a))
    create_proc(2, fn_addr(process_b))

    println("Preemptive multitasking:")

    // Shell loop
    let mut running = 1
    while running == 1 { asm!("wfi") }
    0
}

fn main() -> i64 { 0 }
```

### Output

```
Preemptive multitasking:
AAAAAAAAABBBBBBBBBBAAAAAAAAABBBBBBBB...
```

### Key concepts

- **Context switch**: save/restore all registers via exception frame (272 bytes)
- **SAVE_CONTEXT/RESTORE_CONTEXT**: generated by compiler in vector stubs
- **sched_get_saved_sp()**: reads SP saved by vector stub at 0x47001018
- **sched_set_next_sp()**: triggers SP switch at 0x47001010
- **Round-robin**: each process gets 10ms quantum, then switches
- **fn_addr()**: gets link-time address of a function

## Chapter 5: Syscalls

User processes communicate with the kernel via `svc()`.

### Key code

```fajar
// In the kernel (exception handler):
fn fj_exception_sync(esr: i64, elr: i64, far: i64) {
    let ec = (esr >> 26) & 0x3F
    if ec == 0x15 {  // SVC from AArch64
        // Advance ELR past SVC instruction
        asm!("mrs x0, ELR_EL1")
        asm!("add x0, x0, #4")
        asm!("msr ELR_EL1, x0")

        let syscall_num = syscall_arg0()
        let arg1 = syscall_arg1()

        if syscall_num == 1 {  // SYS_WRITE
            putc(arg1)
        }
        if syscall_num == 0 {  // SYS_EXIT
            // terminate + reschedule
        }
    }
}

// In user process:
fn process_c() -> i64 {
    svc(1, 67, 0)   // SYS_WRITE 'C' — prints via kernel
    svc(0, 0, 0)    // SYS_EXIT — terminates
    0
}
```

### Key concepts

- `svc(num, arg1, arg2)`: compiler builtin that generates `SVC #0`
- ESR.EC = 0x15: SVC exception class
- ELR advance: `SVC` sets ELR to the SVC instruction itself; we add 4 to skip it
- `syscall_arg0/1/2()`: read saved registers from exception frame

## Chapter 6: EL0 User Space

Drop user processes to Exception Level 0 (unprivileged).

### Key code

```fajar
fn create_process_el0(pid: i64, entry: i64) {
    // Copy user code to EL0-accessible memory block
    let user_code = 0x40200000 + pid * 4096
    let mut i = 0
    while i < 512 {
        volatile_write(user_code + i, volatile_read(entry + i))
        i = i + 8
    }

    create_process(pid, user_code)

    // Set SPSR to EL0t (unprivileged)
    let frame_sp = stack_top - 272
    sched_write_proc(frame_sp + 256, 0)  // SPSR M[3:0]=0000

    // Set user stack
    asm!("msr SP_EL0, x0", in("x0") user_sp)
}
```

### MMU page table layout

```
Entry 0:     0x40000000  AP=00 (kernel code, EL1 only)
Entry 1:     0x40200000  AP=01 (user code, EL0 accessible)
Entries 64+: 0x48000000+ AP=01 (process stacks, EL0 accessible)
```

### Key concepts

- **SPSR.M = 0**: `eret` drops to EL0 (unprivileged)
- **AP bits**: control access per exception level in page tables
- **SP_EL0**: separate stack pointer for EL0 (set via `msr SP_EL0`)
- **Code copy**: user code must be in AP=01 region (can't share with kernel)
- **SVC from EL0**: same `svc()` builtin, handled by `__exc_sync_lower`

## Chapter 7: IPC (Message Passing)

Processes exchange messages via kernel mailboxes.

### Key code

```fajar
// Sender
svc(4, dest_pid, value)   // SYS_IPC_SEND

// Receiver (blocks if no message)
let msg = svc(5, 0, 0)    // SYS_IPC_RECV
svc(1, msg, 0)            // print received value
```

### Mailbox: 4-message circular queue

```
[0]  count (0-4)
[8]  head (read index)
[16] tail (write index)
[32] msg[0]: sender(8) + value(8)
[48] msg[1]: sender(8) + value(8)
[64] msg[2]: sender(8) + value(8)
[80] msg[3]: sender(8) + value(8)
```
