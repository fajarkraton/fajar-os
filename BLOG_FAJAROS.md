# FajarOS: The First Operating System Written 100% in Fajar Lang

> *Building an OS kernel, scheduler, syscalls, and IPC — all in one language, with compiler-enforced safety.*

---

## What is FajarOS?

FajarOS "Surya" is an operating system written entirely in [Fajar Lang](https://github.com/fajarkraton/fajar-lang), a statically-typed systems programming language designed for embedded ML and OS integration. The kernel, drivers, scheduler, syscalls, and shell are all written in `.fj` files — no C, no Rust, no assembly (except minimal inline `asm!` for hardware register access).

**Key Stats:**
- **Language:** 100% Fajar Lang
- **Target:** ARM64 (aarch64-none bare-metal)
- **Platform:** QEMU virt (Cortex-A76) / Radxa Dragon Q6A (QCS6490)
- **Kernel size:** ~150 KB ELF
- **Features:** MMU, preemptive scheduling, 10 syscalls, IPC, 65+ shell commands

## How It Works

### Boot Sequence

1. **UEFI/QEMU** loads `fajaros.elf` at 0x40000000
2. **Startup assembly** (generated by Fajar Lang compiler): set stack, VBAR_EL1, call `kernel_main`
3. **MMU init**: 2MB block entries, identity-map MMIO + kernel, enable caches
4. **GICv3 init**: distributor, redistributor, CPU interface, unmask IRQs
5. **Timer init**: 10ms quantum (625K ticks at 62.5MHz), PPI 30 enabled
6. **Shell loop**: prompt → read UART → execute command → WFI

### Preemptive Scheduling

The ARM Generic Timer fires every 10ms (100 Hz). The IRQ handler:

```fajar
fn fj_exception_irq() {
    let irq = gic_ack()
    if irq == 30 {
        timer_set(625000)          // rearm
        sched_switch()             // round-robin
    }
    gic_eoi(irq)
}
```

Context switch is handled by the exception vector stub:
1. `SAVE_CONTEXT`: push x0-x30, ELR_EL1, SPSR_EL1 (272 bytes)
2. Call handler
3. Check context switch flag at 0x47001010
4. If set: `mov sp, new_sp` → switch to next process
5. `RESTORE_CONTEXT` + `eret`

### Syscalls

Processes call the kernel via `svc(num, arg1, arg2)`:

```fajar
// Process C: print via syscall
svc(1, 67, 0)    // SYS_WRITE: print 'C'
svc(0, 0, 0)     // SYS_EXIT: terminate
```

The `svc()` builtin is an assembly stub: `svc #0; ret`. The sync exception handler dispatches based on syscall number.

ELR advance: `asm!("mrs x0, ELR_EL1") asm!("add x0, x0, #4") asm!("msr ELR_EL1, x0")` — skip past the SVC instruction on return.

### IPC

4-message circular queue per process:

```fajar
// Sender
svc(4, 1, 83)     // SYS_IPC_SEND: send 'S' to PID 1

// Receiver
let msg = svc(5, 0, 0)  // SYS_IPC_RECV: blocks until message
svc(1, msg, 0)          // print received character
```

## Compiler Features Used

| Feature | How Used |
|---------|----------|
| `println("text")` | String literals in @kernel → `.rodata` section |
| `return expr` | Early return from functions (value + void) |
| `svc(num, arg1, arg2)` | Syscall via assembly stub |
| `fn_addr(func_name)` | Get function entry address at link time |
| `volatile_read/write` | MMIO register access |
| `asm!("msr REG, x0")` | System register access |
| `timer_count()` | Read physical counter (CNTPCT_EL0) |
| `gic_ack()` / `gic_eoi()` | Interrupt controller (ICC_IAR1_EL1) |
| `mmu_enable(mair, tcr, ttbr0)` | Enable MMU with page tables |
| `sched_read/write_proc()` | IRQ-safe memory access |

## Build & Run

```bash
# Build Fajar Lang compiler
cd ~/fajar-lang
cargo build --release --features native

# Build FajarOS
cd ~/fajar-os
fj build kernel/boot/start.fj --target aarch64 --no-std -o fajaros.elf

# Run on QEMU
qemu-system-aarch64 \
  -M virt,gic-version=3 \
  -cpu cortex-a76 \
  -m 512M \
  -kernel fajaros.elf \
  -device loader,file=fat.img,addr=0x44000000 \
  -nographic
```

## EL0 User Space

FajarOS now runs user processes at ARM64 Exception Level 0 (unprivileged),
with MMU-enforced page table isolation:

```
EL0 (User)                      EL1 (Kernel)
┌────────────────┐              ┌──────────────────┐
│ user_func()    │              │ Syscall handler  │
│ svc(1,'U',0) ──│──SVC #0────→│ putc('U')        │
│ svc(0,0,0)   ──│──SVC #0────→│ EXIT + schedule  │
└────────────────┘  ←──eret──── └──────────────────┘

Page tables: Entry 0 AP=00 (kernel only), Entry 1 AP=01 (user accessible)
```
